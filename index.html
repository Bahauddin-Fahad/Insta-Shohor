Q&A Section

JavaScript is a High Level Interpreted Language. JavaScript executes only one command at a time and follows a specific order. That means, it goes to the next line only when the current line has been finished executing. That is why JavaScript is also called Synchronous Single-Threaded Language. When we run a javascript program, an Execution Context is created which consists of two components, Memory Component and Code Component. First, it allocates or creates memory in the memory component for all the variables and functions that have been declared in the Javascript. Then it starts setting the values of the variable and executes the JavaScript code.

JavaScript is a single threaded language. All the execution processes follow the last in first out mechanism through the call stack. In case of executing a time costly process or operation JavaScript offers Asynchronous Function. The Asynchronous Function avoids the blocking behavior of an application while executing a large process. Because of the asynchronous function, JavaScript single thread doesn't get blocked. The thread sends the asynchronous functions to the Web API or C++ API to handle the large operation, which may take much time while executing. After finishing the functions asynchronously, the functions are sent to the Call Back Queue. To send the functions from the Call Back queue to the Call Stack, a system is needed which follows the First In First Out Mechanism. Event loop is the system that sends the functions stored in the Call Back Queue to the Call Stack. After sending to the Call Stack, the process will be completed.

Local Storage and Session Storage are the storages of a browser which stores the data of the clients.
Local Storage stores the data without any expiration date or conditions. The data expires only when it is deleted manually. Local Storage stores the data for the whole browser.
While in Session Storage, the data is stored in the storage till the browser is opened. Once the browser is closed, the data from the session storage expires. Session Storage stores the data for a particular tab or window. If multiple tabs or windows are opened with the same URL, different session storages will be created for each tab or window.
